noJS.config

说明：

参数配置可以直接写在页面上；也可以通过script标签的data-config属性设置； 还可以将配置独立成一个文件如conf.js，通过script标签的data-config="conf.js"引入即可， 该文件是通过默认获取的基址目录引入的，上级目录表示为../conf.js， 外部引入的配置文件后缀必须是'.js'，且必须执行一次noJS.config方法

使用：
options选项：
参数名 类型 描述 默认值
base String 基址路径，默认会根据noJS.js的文件路径获取
如'js/lib/nojs/noJS.js'对应的默认基址路径为'js/lib/'
global String|Array 设置全局模块，单模块可直接使用字符串
注：全局模块之间可相互依赖，但是不可依赖其他模块 ；只能设置一次，多次无效
pack Object 打包配置
fix String 模块后缀名，一般不用更改 '.js'
update String|Array 设置需要更新浏览器缓存的模块 'all'标示更新所有模块

注1：如果参数配置比较简单，可直接写入script标签的data-config属性

注2：如果配置比较多，可以通过引入外部配置文件的方式统一管理

为了快速获取配置，在引入noJS.js的script上最好加上id="nojs"，否则会读取head中最后一个script。获取该script后，'src'中文件所在目录的上一级会作为默认的base目录
